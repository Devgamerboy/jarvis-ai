# Security Policy

## Supported Versions

Only the latest stable release of JARVIS receives security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We take security vulnerabilities seriously. If you discover a security issue,
please follow these steps:

1. **Do not** open a public GitHub issue.
2. Send a private report via email to the project maintainer, or
   use the **Report a vulnerability** feature under the repository's
   **Security** tab on GitHub.
3. Include as much detail as possible:
   - Affected version(s)
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if known)

You can expect an acknowledgment within 48 hours, and a detailed response
within 5 business days. We will keep you informed of the remediation process.

## Disclosure Policy

We follow a coordinated disclosure process:

1. The report is received and acknowledged.
2. A fix is developed and tested privately.
3. The fix is released in a new patch version.
4. The vulnerability is publicly disclosed after users have had time to update.

## Preferred Languages

We prefer all communications in English.
